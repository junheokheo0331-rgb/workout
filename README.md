# Autoreg — RIR 기반 자동조절 트레이닝 PWA

## 파일
- `index.html` — 앱 화면
- `app.js` — 전체 로직 (RPE 표, 자동조절 엔진, 타이머, 화면)
- `sw.js` — 서비스 워커 (오프라인 캐시)
- `manifest.json`, `icon-*.png` — PWA 설치용

모든 데이터는 **기기의 localStorage**에만 저장된다. 서버로 나가는 데이터는 없다.

## 폰에 설치하는 법 (PWA)
서비스 워커는 `https://` 또는 `localhost`에서만 동작한다. 파일을 그냥 열면(`file://`)
앱은 작동하지만 오프라인 설치는 안 된다. 셋 중 하나를 택한다.

**1) GitHub Pages (가장 간단, 무료)**
1. GitHub에 새 저장소를 만들고 이 폴더의 5개 파일을 그대로 올린다.
2. Settings → Pages → Source를 `main` 브랜치 `/ (root)`로 지정.
3. 몇 분 뒤 `https://<아이디>.github.io/<저장소>/` 주소가 생긴다.
4. 폰에서 그 주소를 연다.
   - **Android/Chrome**: 주소창 메뉴 → "앱 설치" 또는 "홈 화면에 추가"
   - **iPhone/Safari**: 공유 버튼 → "홈 화면에 추가"
5. 홈 화면 아이콘으로 실행하면 주소창 없는 전체화면 앱으로 뜬다.

**2) 로컬 테스트**
```bash
cd 이폴더
python3 -m http.server 8080
# 같은 와이파이의 폰에서 http://<PC_IP>:8080 접속
```

**3) Netlify Drop** — netlify.com/drop 에 폴더를 끌어다 놓으면 즉시 https 주소가 나온다.

## 백그라운드 타이머에 대한 정직한 설명
휴대폰이 브라우저를 완전히 잠재우면 어떤 웹앱도 코드를 계속 실행할 수 없다. 이건
브라우저의 제약이지 구현 문제가 아니다. 그래서 이 앱은 이렇게 처리한다.

- 휴식 타이머는 남은 초를 세지 않고 **끝나는 시각(timestamp)** 을 저장한다.
- 화면을 다시 켜거나 앱으로 돌아오면 현재 시각과 비교해 **정확한 남은 시간을 복원**한다.
- 이미 지났으면 그 즉시 알림·진동·소리로 알린다.
- 앱이 살아 있는 동안에는 정시에 알림이 뜬다.
- 설정에서 **화면 꺼짐 방지(Wake Lock)** 를 켜두면 세트 사이에 화면이 꺼지지 않아 가장 확실하다.

즉 "폰을 주머니에 넣어도 시간은 절대 틀리지 않지만, 화면이 꺼진 채로 정확히 그 순간
알람이 울리는 것은 보장되지 않는다".

## 데이터 백업
설정 → 백업 내보내기로 JSON 파일을 저장할 수 있다. 브라우저 저장소를 지우거나
기기를 바꾸면 기록이 사라지므로 가끔 내보내 두는 것을 권한다.
